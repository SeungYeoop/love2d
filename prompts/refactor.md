# Refactor Prompt

먼저 `AI_WORKFLOW.md`, `PROJECT_RULES.md`, `TASK.md`를 읽고 아래 규칙을 지키면서 LÖVE2D 리팩토링 계획 또는 최소 범위 리팩토링을 진행해줘.

- 먼저 현재 구조를 어떻게 이해했는지 짧게 정리해줘.
- 큰 수정이 필요하면 바로 실행하지 말고, 변경 이유 / 대상 파일 / 예상 영향 / 롤백 포인트를 먼저 보여주고 승인을 받아줘.
- 기존 동작은 유지해야 한다.
- 한 파일에 코드를 몰아넣지 말고, 기능별로 분리하는 방향을 우선해줘.
- 요청 범위를 벗어나는 스타일 수정이나 대규모 정리는 하지 말아줘.
- 수정 전에는 어떤 파일을 왜 바꾸는지 설명하고, 수정 후에는 변경 내용을 요약해줘.
- `main.lua`가 맡는 책임을 줄이되, LÖVE 콜백 연결은 깨지지 않게 해줘.
- 입력, 상태, 렌더링, 엔티티, 씬 단위로 나눌 수 있으면 우선 제안해줘.
- 코드가 바뀌면 `.\play.cmd`를 실행하거나 watcher 기준으로 결과 확인까지 포함해줘.
- 마지막에는 다음 개선 3개 또는 남은 리스크를 알려줘.
