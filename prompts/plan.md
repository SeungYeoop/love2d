# Plan Prompt

먼저 `AI_WORKFLOW.md`, `PROJECT_RULES.md`, `TASK.md`를 읽고 아직 구현하지 말고 LÖVE2D 기준 계획만 세워줘.

아래 규칙을 지켜줘.

- 내가 요청한 작업을 어떻게 이해했는지 2~4줄로 먼저 요약해줘.
- 바로 코드를 수정하지 말고 계획만 작성해줘.
- 큰 수정일수록 단계별로 나눠서 설명해줘.
- 각 단계마다 수정 대상 파일, 이유, 위험 요소를 적어줘.
- 기존 동작을 바꾸는 지점이 있다면 반드시 따로 표시해줘.
- 한 파일에 기능이 몰리지 않게 어떤 식으로 분리할지 제안해줘.
- 내 승인 전에는 리팩토링, 파일 이동, 대규모 변경을 실행하지 말아줘.
- `main.lua`, `conf.lua`, `src/` 구조 중 어디에 손대는지 분명히 적어줘.
- `love.load`, `love.update`, `love.draw`, 입력 콜백 중 어떤 흐름에 영향이 있는지도 적어줘.
- 이번 계획이 첫 playable prototype을 만드는 단계인지, 후속 개선 단계인지도 적어줘.
