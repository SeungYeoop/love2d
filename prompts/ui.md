# UI Prompt

먼저 `AI_WORKFLOW.md`, `PROJECT_RULES.md`, `TASK.md`를 읽고 아래 규칙을 지키면서 LÖVE2D 화면/UI 수정만 진행해줘.

- 먼저 내가 요청한 UI 변경을 짧게 정리해줘.
- 요구사항이 모호하면 바로 구현하지 말고, 이해한 방향을 먼저 확인해줘.
- 기능 로직은 불필요하게 건드리지 말고, UI 관련 수정만 최소 범위로 진행해줘.
- 수정 영향이 크면 먼저 계획만 제안하고 내 승인을 받아줘.
- 파일을 바꾸기 전에는 어떤 파일을 왜 수정하는지 설명해줘.
- 한 파일에 모든 UI 코드를 몰아넣지 말고, 역할 분리가 필요하면 제안만 먼저 해줘.
- 버튼, HUD, 타이틀 화면, 점수 표시 같은 LÖVE2D 화면 요소 기준으로 생각해줘.
- 가능하면 `love.draw`를 과하게 비대하게 만들지 않는 방향을 우선해줘.
- 수정이 끝나면 `.\play.cmd`를 실행하거나 watcher 기준으로 결과를 확인해줘.
- 마지막에는 다음 UI 개선 3개 또는 필요한 질문을 짧게 정리해줘.
