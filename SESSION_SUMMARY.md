# SESSION_SUMMARY

## 현재 세션 요약
- 이 저장소를 LÖVE2D용 AI 프로토타이핑 템플릿으로 강화하는 중이다.
- 기존 소스코드는 유지하고, 문서와 프롬프트, 실행 보조 스크립트를 정리하고 있다.
- 현재 확인된 코드 파일은 `main.lua` 하나다.

## 현재 프로젝트에 대해 파악한 점
- 언어는 Lua다.
- `love.load`, `love.update`, `love.draw` 사용으로 보아 LÖVE2D 프로젝트다.
- 현재는 템플릿 또는 초기 프로토타입 시작점으로 보기 적절하다.
- 아직 구조 분리나 예제 모듈 추가는 하지 않았다.
- 이제 목표는 "짧은 명령으로 만족도 높은 프로토타입"과 "완료 후 즉시 실행 확인"까지 포함한다.

## 이번에 만든 운영 기준
- 큰 수정 전 승인
- 기능별 분리 우선
- 막히지 않으면 합리적 가정으로 먼저 진행
- 기존 동작 유지
- 최소 수정 원칙
- 수정 전/후 변경 이유 설명
- 에러는 원인 분석 후 수정
- 첫 결과물은 playable prototype 우선
- 작업 후 LÖVE2D 재실행 또는 watcher 확인
- 각 라운드 종료 시 다음 개선안 제안
- `README.md`는 사용자용, 나머지 운영 문서는 AI용으로 분리

## 다음 세션에서 바로 참고할 사항
- 리팩토링이나 구조 변경은 아직 승인되지 않았다.
- 당분간 `main.lua`의 기존 동작을 유지해야 한다.
- 구조 개선이 필요해 보여도 먼저 계획 제안만 해야 한다.
- 문서는 LÖVE2D 템플릿 기준으로 유지한다.
- 프롬프트 템플릿은 첫 구현용과 반복 개선용으로 나뉜다.
- 자동 실행은 `play.cmd` / `watch.cmd` 흐름을 우선 사용한다.

## 사용자 의도 메모
- AI가 작업 범위를 벗어나지 않길 원함
- 한 파일에 모든 코드가 쌓이는 것을 원하지 않음
- 재사용 가능한 프롬프트와 세션 문맥 정리를 원함
- 이 프로젝트를 LÖVE2D용 바이브 코딩 템플릿으로 쓰고 싶어 함
- README는 사용자 튜토리얼 역할을 하길 원함
- 첫 요청만으로도 구조 있는 프로토타입을 만들고 싶어 함
- AI가 다음 개선 방향을 주도적으로 추천하길 원함
- 결과가 나올 때마다 LÖVE2D가 자동으로 다시 열리길 원함
